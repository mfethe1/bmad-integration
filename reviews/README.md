# Adversarial Review Standards

This directory contains the rules and prompts for Adversarial Reviews.

## The Core Mandate
Reviewers *must* find issues. A review with zero findings is unacceptable and triggers a halt.

## The "Looks Good" Ban
The phrase "looks good" (and its variants) is strictly forbidden. 
You must assume problems exist and find them. Look for edge cases, security vulnerabilities, scale limitations, and missed acceptance criteria.

## Information Asymmetry
Reviews are executed with fresh context. The reviewing agent evaluates the artifact itself, not the original developer's intent or reasoning.
